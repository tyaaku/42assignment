o・DebianまたはRockyを使用
o・グラフィカルインターフェースを使用しない。
・サーバー起動時にSELinuxが実行される。
o・AppArmorも実行される必要がある（Debianの場合）
o・LVMのパーティションが２つ以上ある。
・審査の際に選択したOS周りについて説明できる。
o・SSHサービスが4242ポートでのみ実行される。
o・SSHを使用した際にrootユーザーとしては接続できない。
o・UFWを使用してポート4242だけが開かれた状態にする必要がある。
o・仮想マシン起動時にファイアウォールがオンになって必要がある。
o・ホスト名がintra名の末尾に42をつけたものになっている。
o・sudoをインストールする
o・rootユーザーだけでなくintra名だけのユーザーも作る
o・そのユーザーを<user42><sudo>の２つのグループに参加させる。
o・強力なパスワードポリシーを作成する。
{
	o・パスワードは３０日後に期限切れにする必要がある。
	o・パスワードは最低でも２日は変更できません。
	o・パスワードの有効期限が切れる７日前に警告メッセージを送ります。
	o・パスワードは少なくとも10文字じゃないとダメ、大文字、小文字、数字を含まないとダメ、
	３つより多く同じ文字を連続させたらアウト（aaaa4242とかダメ）。
	o・パスワードにはユーザーの名前を含めてはいけない。
	・次のルールはrootパスワードには適用されない。→以前のパスワードに含まれていない文字を少なくとも７つは持ってないといけない。
	・一個上のルール以外はrootパスワードも満たしている必要がある。
}
・sudoグループの強力な設定をする。
{
	・sudoを使用した認証は、パスワードが３回間違えた場合に制限される必要がある。
	・sudoを使用してパスワードが間違って入力された場合に、独自のメッセージを表示する必要がある。
	・sudoを使用した各アクションは、入力および出力の両方がアーカイブされる必要があります。
	ログファイルは/var/log/sudoフォルダに保村されないといけない。
	・セキュリティの理由から、TTYモードを有効にする必要がある。
	・同様のセキュリティ理由から、sudoが使用できるパスは制限される必要がある。
	/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin
	ここだけ
}
・monitoring.shというスクリプトを作成する。
{
	このスクリプトはbashで開発される必要がある。サーバーの起動時に、スクリプトは１０分ごとに、全ての端末に以下の情報を表示する。
	バナーは任意、エラーは見えてはいけない、スクリプトは常に以下の情報を表示できる必要がある。
	・OSのアーキテクチャとそのカーネルバージョン
	・物理プロセッサの数
	・仮想プロセッサの数
	・サーバー上の現在の利用可能なRAMとその利用率（％）
	・サーバー上の現在の利用可能なメモリとその利用率（パーセンテージ）。
	・プロセッサの現在の利用率（パーセンテージ）。
	・最後の再起動の日付と時刻。
	・LVMがアクティブかどうか。
	・アクティブな接続の数。
	・サーバーを使用しているユーザーの数。
	・サーバーのIPv4アドレスとそのMAC（Media Access Control）アドレス。
	・sudoプログラムで実行されたコマンドの数。
}
